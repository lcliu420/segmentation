# 胃部医学图像分割 Baseline 项目说明

## 1. 项目概述

本目录用于搭建一个面向论文实验的胃部医学图像分割 baseline。

当前已有两种模态数据：

- `WL`
- `NBI`

两种模态已经分开整理，并且已经完成固定的数据划分。第一阶段不做多模态融合，不做复杂网络改进，先建立一个标准、可复现、可对比的基础实验流程。

本阶段的核心目标是完成两组独立实验：

1. `U-Net + WL only`
2. `U-Net + NBI only`

后续所有模型改进、损失函数改进、模态融合方法、模块设计和论文对比，都应建立在这套 baseline 之上。

## 3. Baseline 的实验目标

第一阶段使用标准 `U-Net` 作为分割基线模型，完成以下任务：

- 为 `WL` 模态单独训练一个分割模型
- 为 `NBI` 模态单独训练一个分割模型
- 在各自测试集上独立评估性能
- 输出论文中可以直接使用的 baseline 指标结果

这一阶段不追求复杂创新，重点是：

- 跑通完整训练流程
- 保证实验可复现
- 为后续改进方法提供可靠参照

## 4. 基线实验的基本设定

建议采用以下基础设置作为第一版 baseline：

- 模型：标准 `U-Net`
- 输入：RGB 胃镜图像
- 输出：二分类分割 mask
- 训练方式：单模态独立训练
- 任务类型：二分类语义分割

建议优先记录的指标：

- `Dice`
- `IoU`
- `Precision`
- `Recall`

建议优先尝试的损失函数：

- `BCEWithLogitsLoss`
- `Dice Loss`
- `BCE + Dice`

第一版 baseline 可以先从最稳妥的组合开始，例如：

- 损失函数：`BCE + Dice`
- 最优模型选择指标：验证集 `Dice`

## 5. 后续完整执行路线

下面按“从现在到实验完成”的顺序，列出本项目后续需要完成的所有主要步骤。

### 第 1 步：整理训练工程结构

需要在当前目录下搭建基础工程结构，建议至少包含：

- `datasets/`
- `models/`
- `utils/`
- `outputs/`
- `train.py`
- `test.py`
- `requirements.txt` 或环境说明

这一阶段的目标是让目录结构清晰，后续训练、测试、保存结果都有固定位置。

### 第 2 步：实现数据读取模块

需要完成一个标准分割任务数据集读取器，至少包括：

- 读取 `images/` 和 `masks/`
- 根据文件名进行配对
- 对图像和 mask 做相同的空间变换
- 将 mask 转换为训练可用的二值标签
- 支持 `train / val / test` 三种模式

这一阶段还需要做数据读取正确性检查，例如：

- 图像尺寸是否一致
- mask 是否真的是二值图
- 数据增强是否只在训练集启用
- 图像和 mask 是否对齐

### 第 3 步：实现预处理与增强

需要确定并实现基础预处理流程，例如：

- 图像 resize 到统一尺寸
- 图像归一化
- mask 二值化

训练集可选的数据增强包括：

- 随机水平翻转
- 随机垂直翻转
- 随机旋转
- 颜色扰动

注意：

- baseline 阶段增强不要太复杂
- 验证集和测试集不应使用随机增强

### 第 4 步：实现标准 U-Net 模型

需要补充一个可直接训练的标准 `U-Net`，包括：

- 编码器
- 解码器
- skip connection
- 最终二分类输出层

这一阶段的要求不是追求最复杂实现，而是保证：

- 结构标准
- 代码清晰
- 训练稳定
- 后续容易替换成更强模型

### 第 5 步：实现训练脚本

训练脚本至少应支持以下功能：

- 指定模态：`WL` 或 `NBI`
- 自动读取对应的 `train / val` 数据
- 配置 batch size、epoch、learning rate
- 计算训练损失
- 在验证集上评估指标
- 保存最优模型
- 输出训练日志

训练脚本建议至少保存以下内容：

- 每个 epoch 的训练损失
- 每个 epoch 的验证 `Dice`
- 最优模型权重
- 训练参数配置

### 第 6 步：先完成 WL baseline

第一组正式实验建议先跑 `WL only`：

- 使用 `sorted_075/WL/train`
- 使用 `sorted_075/WL/val`
- 根据验证集选最优模型
- 在 `sorted_075/WL/test` 上评估

需要产出：

- `WL` 的测试集指标
- 最优模型权重
- 若干预测可视化结果

### 第 7 步：再完成 NBI baseline

第二组正式实验跑 `NBI only`：

- 使用 `sorted_075/NBI/train`
- 使用 `sorted_075/NBI/val`
- 根据验证集选最优模型
- 在 `sorted_075/NBI/test` 上评估

需要产出：

- `NBI` 的测试集指标
- 最优模型权重
- 若干预测可视化结果

### 第 8 步：实现测试与评估脚本

需要单独的测试脚本，负责：

- 加载最优模型
- 在测试集上逐张推理
- 计算整体平均指标
- 保存预测 mask
- 可选保存可视化对比图

建议可视化内容包括：

- 原图
- 真值 mask
- 预测 mask
- 叠加图

### 第 9 步：整理实验输出结果

建议统一输出到 `outputs/` 下，例如：

```text
outputs/
├── wl_unet/
│   ├── checkpoints/
│   ├── logs/
│   ├── predictions/
│   └── metrics.json
└── nbi_unet/
    ├── checkpoints/
    ├── logs/
    ├── predictions/
    └── metrics.json
```

这一阶段的目标是让每组实验的结果独立保存，方便复查、对比和论文整理。

### 第 10 步：完成论文可用的结果汇总

当 `WL` 和 `NBI` 两组 baseline 都跑完后，需要整理出论文可直接使用的内容：

- 每组实验的测试集平均指标
- 典型样本可视化结果
- 训练和验证曲线
- 训练设置表格
- baseline 对比表

最终应至少形成一张类似如下的结果表：

| 方法 | 模态 | Dice | IoU | Precision | Recall |
| --- | --- | ---: | ---: | ---: | ---: |
| U-Net | WL | - | - | - | - |
| U-Net | NBI | - | - | - | - |
